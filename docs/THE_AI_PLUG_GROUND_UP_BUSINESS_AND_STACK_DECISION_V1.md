## Executive decision

The project is a **connected local media + lead + business systems** company.

| Layer | Brand / role |
|-------|----------------|
| Visitor / media / lead / advertising | **Where To Go SA** — public San Antonio visitor guide and proof-of-work |
| Business systems + practical AI adoption | **The AI Plug** — owner-facing systems, intake, follow-up, CRM-lite, ads support |
| Client delivery (test first) | **GoHighLevel or similar** — CRM, forms, missed-call text-back, review requests, subaccounts |
| Internal proof + premium custom | **Custom dashboard** — Where To Go SA owner tools, prospecting, lead review, future premium builds |

**Best ground-up path:**

1. **Where To Go SA** = visitor / media / lead / advertising brand  
2. **The AI Plug** = business systems + AI adoption brand  
3. **GoHighLevel or similar** = test client delivery stack (do not hard-lock until proven)  
4. **Custom dashboard** = internal proof + premium custom future layer  
5. **Home Services Lead Rescue** = first sales wedge  
6. **Restaurants / Hospitality** = second monetization lane (Where To Go SA visibility)  
7. **Transportation / Concierge** = third monetization lane

## Stack decision

**Principle:** Use the right stack for the right job.

### Public site stack — keep

| Component | Use |
|-----------|-----|
| GitHub | Source control |
| Netlify | Hosting, functions where needed |
| Static / public pages | Visitor guide, SEO, business pages, forms where appropriate |
| Where To Go SA content | Visitor-first pages, Ask A Local proof, transportation requests |

**Do not** move the public visitor guide to a heavy CMS or client portal stack.

### Internal proof stack — keep custom

| Component | Use |
|-----------|-----|
| Where To Go SA owner dashboard | Daily brief, lead review, restaurant tools |
| Restaurant prospecting | Sales Mode, research, proposals (internal) |
| Transportation requests | Visitor intake → internal review |
| Custom proof-of-work | Where To Go SA as live example for prospects |
| Future custom dashboard demos | Premium / complex clients only |

**Do not** rebuild completed dashboard modules unless fixing bugs or intentional redesign (`DASHBOARD_STATUS.md`).

### Client delivery stack — test before committing

**Default hypothesis (not locked):**

| Need | Preferred tool |
|------|----------------|
| CRM / pipelines | GoHighLevel (or similar) |
| Forms | GHL |
| Missed-call text-back | GHL |
| Review requests | GHL |
| Follow-up sequences | GHL |
| Client subaccounts | GHL |
| Paid ads | Google / Meta / TikTok directly |
| Custom build | Only when GHL cannot do it **or** client pays premium |

**Critical rule:** Do **not** hard-lock GoHighLevel until a stack test proves it is **better, cheaper, faster, and safer** than custom or hybrid delivery.

## 7-day GoHighLevel test

Run a real client-shaped test (sandbox or pilot) before standardizing delivery.

| Test item | Pass criteria |
|-----------|----------------|
| Missed-call text-back | Works with disclosure and owner approval |
| Form → pipeline | Lead lands in correct stage |
| Owner alert | Owner notified on new lead |
| Review request | Compliant request flow |
| Three-message follow-up | Sequence works; opt-out honored |
| Simple landing page | Mobile-usable; tracks source |
| Monthly report | Owner-readable summary export or view |
| SMS compliance flow | Consent, opt-out, no auto-send without approval |
| Actual cost per client | Documented monthly tool cost at 1, 3, 5 clients |

**Outcome:** Document go / no-go / hybrid in `TECH_STACK_STATUS.md` after test completes (separate tech task).

## Compliance / guardrails

Every offer involving SMS, email, or follow-up **must** include:

- Owner approval before customer-facing automation
- Consent language (visible on forms)
- Opt-out language
- No auto-send without approval
- Call recording disclosure if applicable
- No review gating
- No guaranteed results
- No ad ROI promises
- Client owns ad spend / tools where possible
- No fronting ad spend on owner’s card

### Never promise

- Guaranteed leads, traffic, rankings, revenue, jobs, bookings, or reviews
- Guaranteed ad ROI
- AI replacing staff
- Unsupervised customer-facing automation

### Restaurant / visibility specific

- Reviewed for fit; not listed by default
- Payment does not determine inclusion
- No guaranteed visitor traffic, reservations, or revenue

### Professional services

- AI does not provide legal, medical, tax, financial, or insurance advice
